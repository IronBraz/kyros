# Database

PostgreSQL schema for Kyros. Single source of truth: [`kyros_schema.sql`](kyros_schema.sql).

## Requirements

- **PostgreSQL 18.1+** (uses native `uuidv7()` function and `gen_random_uuid()`)
- Optional: `pgcrypto` extension if your distribution does not enable random UUID generation by default

## Apply the schema

```bash
# Create the database
createdb -U postgres kyros

# Apply schema
psql -U postgres -d kyros -f kyros_schema.sql
```

## Tables

| Table              | Purpose                                                      |
|--------------------|--------------------------------------------------------------|
| `tenants`          | Top-level brand / company                                    |
| `stores`           | Physical store locations (JSONB `settings` for queue config) |
| `entry_points`     | QR / NFC entry locations within a store                      |
| `service_points`   | Physical service stations ("Cassa 1", "Cassa 2", ...)        |
| `sessions`         | Customer queue sessions (ticket lifecycle, heartbeat)        |
| `marketing_cards`  | Content feed shown in waiting room                           |
| `admin_access`     | PIN-based admin access codes for Control Room                |

## Patterns

- **UUIDv7** time-sortable IDs everywhere via `uuid_generate_v7()`
- **Ticket codes** auto-generated by trigger as `A-001`, `A-002`, ..., `B-001`, ... (resets daily at midnight in the store's timezone)
- **JSONB** store settings keep queue and display config flexible
- **Partial indexes** on the hot path (waiting/called sessions) and **GIN** indexes on JSONB

See [`../specs/04_Database.md`](../specs/04_Database.md) for the detailed schema rationale and indexing strategy.

## Utility script

`manage_kyros_db.ps1` — PowerShell helper for common DB ops (init, reset, backup, check). Optional, not required for production.
